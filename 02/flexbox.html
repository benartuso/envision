Two big caveats on our use of CSS here:
  • For adding in the layout styles we're discussing in this document, we're using inline style. This is where style is added directly to an element via the attribute style="property:value; property:value" etc. directly in the opening HTML tag. This is purely to make it easier for you to follow along, and is for demonstration purposes. You should never do this when writing real HTML+CSS! The best practice is to link in an external CSS sheet, which will allow you to keep your content and style separate.
  • Another "do as we say, not as we do" moment: in addition to those styles added in-line, we also have some additional styles in a style tag in the document header. You don't need to pay super close attention to these, because they're mostly cosmetic (adding margins, etc.) to make the document easier to read overall. We mention this here for two reasons:
    • In practice, don't do this! Use an external stylesheet, instead.
    • If you try to recreate some of the examples and they don't look EXACTLY the same, check what styles we're applying! It's nothing critical, but makes a bit of cosmetic difference.

Normal flow

Normal flow is the default way that our browsers arrange content.

As you can see, this happens more or less in order from top to bottom, and in the same order that elements are specified in our HTML markup.

Text in this div is in the middle,

Because that's the order it's specified in our HTML.

Inline vs. block elements

As we discussed last week, most elements have default displays of either inline, where the element sits next to other elements and doesn't take up an entire line, and block, where the element cannot be in line with other elements, as it takes up an entire line.

This paragraph element is block—it starts a new line.

This is one span and this is another. These are inline, as they are able to exist on the same line.

Here's what's new this week: whether an element is block or inline is not a fixed, immutable property of that element type. It's a CSS property, called display, and can be changed like any other CSS property!

Messing with elements' display property:

We've established that paragraphs have block display. All we really mean here is that they have block display by default.

This red paragraph also has its display changed to inline.

Flexbox!

Positioning things in documents, using only the above display properties and the occasional exact, pixel-level specification of where something should go, was a huge pain.

So, an additional display method was developed, called the CSS flexible box model, or Flexbox.

Flexbox helps with the arrangment of multiple items (divs, paragraphs, images, text, etc.) within a container (usually a div).

As we just saw, justify-content is used to align items around the MAIN FLEX AXIS. By default, that is the "row", as we discussed earlier, so our main flex axis goes across the div from left to right. If our flex direction was "column," these properties would do the same thing on the vertical axis instead!

Arranging content on the SECONDARY FLEX AXIS with align-items

By default, our inner divs "stretch" to take up the whole height of the container. There is a bit of space between the top of the inner divs and the edges of the outer divs, but that's only because we put in a slight "margin" property to make things look a bit clearer.

If we change this property to another align-items value, our inner divs will only be as tall as they need to be to display their content, and we can then align them wherever we want vertically.

Changing properties of individual items

We'd be quite limited if all of our flex children needed to take up the same amount of space, as they have in the examples we've discussed so far.

Thankfully, the flex-grow property allows us to specify the relative sizes of the items inside of a flex container.

In contrast to all of the other styles, which have (so far) been applied to the PARENT (flex container), these are applied to the CHILDREN (flex items) inside the parent container, since each will have a different flex-grow factor.
