As you'll see here, pure text can be entered as an element, without being needing to be contained in any element other than the body. You can use the tags discussed in class, such as strong for bolding, or em for italics. As you'll see, though, entering a line break in this document does NOT result in a corresponding line break in the HTML! All the text just strings together, which is a bit unfortunate for those of us who're interested in making readable, well-separated text.

So, how do we effectively separate our text?

1: Using paragraphs

One strategy is to use paragrpahs, the p tag. As we discussed, paragraphs are a BLOCK LEVEL ELEMENT, which means that each paragaph will start its own new line and take up the full width of its container.

As such, when I start a new paragraph, you can see that the text is nicely separated from the paragraph before it.

I can do this as many times as I like, and it will continue to be separated!

2: Using divs

Divs, like paragraphs, are block level elements, but are more general. While a paragraph can (mostly) only hold text, divs can contain lots of things: text, headers, SVG shapes, paragraphs, images, other divs, etc.
Since they're also block level elements, though, they can be used to separate text!
The main difference you'll notice is that they lack the visually appealing padding between each that the paragraphs have.
While we can configure that (and add it to our divs) using CSS properties to make them function more like paragraphs, it's a much better practice to use paragraphs to contain and separate your text, and use divs to structure and organize the various content on your page.

3: Using line breaks (less recommended)

Additionally, you can use the break tag to separate lines without wrapping the text in anything, giving a similar look to the spacing of organizing with divs.
Notice that br is a self-closing tag: it doesn't need an opener and closer, just the single tag.

On attributes:

As we discussed in class, attributes are ways to configure additional details about an HTML element. For example, this line is an a (anchor/link) tag with NO attributes.
The browser doesn't give it an underline or link stylization, and clicking it doesn't do anything. This is because the browser has no idea where it's supposed to link to!
In contrast, this line has the attribute href="https://www.google.com." Now it is stylized as a link, and functions like one too. Click it and check it out!

A similar effect can be shown with images.
